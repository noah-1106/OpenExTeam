## 核心架构原则

**Dashboard 只与主 Agent 通信，不感知子 Agent 存在。**

```
Dashboard ──► Adapter ──► 主 Agent ──► 子 Agent(s)
   ▲           │           │
   └───────────┴───────────┘
        回调统一来自主 Agent
```

所有执行细节（子 Agent 如何创建、如何执行）由各框架底座自行处理。Dashboard 只负责：
1. 发送统一格式的消息
2. 接收主 Agent 的回调
3. 根据回调更新状态

## 三、开开（后端/Adapter）工作清单

### 3.1 Adapter 核心职责（简化版）

**Adapter 只做三件事：**
1. **消息格式转换**：Dashboard 统一格式 ↔ 框架原生格式
2. **发送给主 Agent**：调用框架 API 发送消息给主 Agent（主 Agent 内部自行创建子 Agent）
3. **解析回调**：将主 Agent 的回调转换为统一格式

**Adapter 不做的事：**
- ❌ 不管理子 Agent 生命周期
- ❌ 不维护长连接
- ❌ 不处理框架认证差异（在连接层处理）

**关键原则：**
- Dashboard 只认识"一个 Agent"
- 子 Agent 的存在对各框架透明
- 所有回调来自主 Agent（子 Agent 结果已汇总）

### 3.3 各框架 Adapter 实现

**OpenClaw Adapter：**
```javascript
class OpenClawAdapter {
  // 发送消息给主 Agent
  async send(agentId, message) {
    // 1. 转换消息格式
    const openclawMessage = {
      tool: "sessions_send",
      params: {
        sessionKey: agentId,  // 主 agent session
        message: message.content
      }
    };
    
    // 2. WebSocket 发送给主 Agent
    // 主 Agent 内部通过 sessions_spawn 创建子 Agent 执行
    await this.webSocketSend(openclawMessage);
  }
  
  // 解析主 Agent 回调（子 Agent 结果已汇总）
  parse(raw) {
    return {
      type: raw.type,  // task_complete | workflow_step_complete
      taskId: raw.taskId,
      result: raw.result
    };
  }
}
```

**DeerFlow Adapter：**
```javascript
class DeerFlowAdapter {
  async send(agentId, message) {
    // 直接 HTTP POST 给 lead agent
    await fetch(`${this.endpoint}/api/chat`, {
      method: 'POST',
      body: JSON.stringify({
        message: message.content,
        thread_id: agentId,  // lead agent thread
        mode: 'ultra',  // 内部创建 sub-agents
        callback: message.callback
      })
    });
  }
}
```

**Hermes Adapter：**
```javascript
class HermesAdapter {
  async send(agentId, message) {
    // 通过 api_server.py HTTP API 发送给主 agent
    await fetch(`${this.endpoint}/chat`, {
      method: 'POST',
      body: JSON.stringify({
        prompt: message.content,
        agent: agentId,
        callback: message.callback
      })
    });
    // 主 agent 内部通过 delegate_task 创建子 agent
  }
}

### 3.5 串行推进逻辑伪代码

```javascript
// 工作流启动
async function startWorkflow(jobId) {
  const tasks = await db.tasks.find({ jobId }).sort({ createdAt: 1 });
  const firstTodo = tasks.find(t => t.status === 'todo');
  
  if (!firstTodo) return { error: 'No pending tasks' };
  
  // 发送第一个步骤
  await sendWorkflowStep(jobId, firstTodo, 1, tasks.length);
  
  // 更新 Job 状态
  await db.jobs.update(jobId, { status: 'in-progress' });
  
  return { started: true, firstTaskId: firstTodo.id };
}

// 步骤完成回调
async function onStepComplete(taskId, jobId, result) {
  // 1. 标记当前任务完成
  await db.tasks.update(taskId, { status: 'done' });
  
  // 2. 查找同 Job 的下一个任务
  const tasks = await db.tasks.find({ jobId }).sort({ createdAt: 1 });
  const nextTask = tasks.find(t => t.status === 'todo');
  
  if (nextTask) {
    // 3. 自动启动下一步
    const stepIndex = tasks.indexOf(nextTask) + 1;
    await sendWorkflowStep(jobId, nextTask, stepIndex, tasks.length);
  } else {
    // 4. 全部完成
    await db.jobs.update(jobId, { status: 'done' });
    await notifyWorkflowComplete(jobId);
  }
}

// 发送步骤消息
async function sendWorkflowStep(jobId, task, stepIndex, totalSteps) {
  const message = {
    type: 'workflow_step',
    jobId,
    taskId: task.id,
    stepIndex,
    totalSteps,
    content: {
      title: task.title,
      description: task.description,
      context: `步骤 ${stepIndex}/${totalSteps}`
    }
  };
  
  await adapter.send(task.agent, message);
  await db.tasks.update(task.id, { status: 'in-progress' });
}
```

## 四、关键设计决策

### 4.1 消息总线架构
- **Dashboard 只与主 Agent 通信**，子 Agent 对 Dashboard 完全透明
- **Adapter 只做格式转换**，不管理子 Agent 生命周期
- **所有回调来自主 Agent**，子 Agent 执行结果由主 Agent 汇总后返回

### 4.2 工作流推进
- 串行执行，高频调度实现吞吐
- Dashboard 收到回调后自动触发下一步
- 失败暂停，人工介入后可恢复

### 4.3 框架绑定
- OpenClaw：WebSocket RPC，需 Tailscale（远程）或 localhost（本地）
- DeerFlow：原生 HTTP REST，最简单
- Hermes：api_server.py HTTP API 或 SSH+CLI

## 七、疑问解答

**Q: 子 Agent 怎么创建的？**
A: 各框架底座自行处理，Dashboard 不感知：
- OpenClaw: `sessions_spawn`
- DeerFlow: `mode=ultra` 内部创建
- Hermes: `delegate_task` 工具

**Q: 工作流失败了怎么办？**
A: 任务失败后暂停工作流，通知对应 Agent 和 Dashboard，人工介入后重新启动。

**Q: OpenClaw 远程连接怎么配？**
A: 服务器安装 Tailscale，Dashboard 连接 `wss://oc-server.tailnet.ts.net:18789`，执行一次 `openclaw devices approve` 完成配对。

**Q: 一个 Job 的任务顺序怎么定？**
A: 按 `createdAt` 创建时间排序，创建任务时按顺序创建。
